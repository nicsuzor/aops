# Visual Analysis Protocol

A dedicated protocol for critically evaluating visual artifacts (like UI mockups, screenshots, or rendered data visualizations) to ensure they adhere to core UI/UX principles. This protocol must be applied during Qualitative Assessment when UI changes or visual artifacts are involved.

## Evaluation Criteria

When conducting a visual analysis, you must explicitly evaluate the following dimensions:

### 1. Legibility

Can the text actually be read? Look beyond mere presence and evaluate usability.

- Are there overlapping labels?
- Is text crushed or truncated awkwardly?
- Are there contrast failures between text and its background?

### 2. Information Density

Is the user overwhelmed by visual noise? Evaluate the cognitive load imposed by the visual presentation.

- Are there too many elements displayed at once (e.g., a giant mosaic of tiny, blank rectangles)?
- Does the interface lack appropriate aggregation or roll-up mechanisms for large datasets?
- Is there sufficient whitespace to group related information and provide visual rest?

### 3. Spatial Metaphor

Does the layout physically represent the data structure accurately?

- Do the sizes, positions, and groupings of visual elements correctly reflect their underlying relationships and importance?
- If the layout distorts the data to fit a specific shape (e.g., hiding text to fix vertical distortion), does it compromise the fundamental understanding of the data?

## Execution

When instructed to perform visual analysis, do not merely state what is present in the image (e.g., "there are boxes on a screen"). You must critically evaluate the image against the criteria above. Look for nuance, identify potential usability issues, and assess the impact on the user experience.
